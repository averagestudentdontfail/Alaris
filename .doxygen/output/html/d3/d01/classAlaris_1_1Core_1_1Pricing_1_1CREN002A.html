Detailed Description

Unified American option pricing engine supporting all rate regimes.

Key improvements over base CREN001A:

  • ASINH grid distribution - higher density near ATM, sparser at boundaries
  • Neumann boundary conditions (gamma = 0) - more stable for extreme prices
  • Regime detection for positive/negative rate handling

Applies Neumann boundary conditions (gamma = 0).

Neumann conditions enforce that the second derivative (gamma) is zero at boundaries. This is more stable than Dirichlet conditions and reflects that option value becomes linear in the far wings.

At left boundary (S→0): V[0] = 2*V[1] - V[2] At right boundary (S→∞): V[N-1] = 2*V[N-2] - V[N-3]

Builds ASINH-distributed grid with higher density near spot.

Uses QuantLib/kwinto-cuda sinh interpolation formula:

  1. Define grid bounds: xMin = xMid - scale*σ*√T, xMax = xMid + scale*σ*√T
  2. Transform to y-space: yMin = asinh((xMin-xMid)/density), yMax = asinh((xMax-xMid)/density)
  3. Linear interpolate in y-space: y = yMin*(1-ξ) + yMax*ξ
  4. Transform back: x = xMid + density * sinh(y)

This concentrates grid points near xMid where option value curves sharply. References: fdmblackscholesmesher.cpp, kwFd1d.cpp

Builds tridiagonal system with variable grid spacing.

Black-Scholes PDE in log-spot space: ∂V/∂t + (r-q-σ²/2)∂V/∂x + (σ²/2)∂²V/∂x² - rV = 0

PDE coefficients matching kwinto-cuda/QuantLib:

  • a0 = -r (kill/discount term)
  • ax = r - q - σ²/2 (drift/convection)
  • axx = σ²/2 (diffusion)
